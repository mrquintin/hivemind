================================================================================
RESIDUAL RISKS
================================================================================

1) DuckDuckGo rate limiting: the duckduckgo-search library relies on
   DuckDuckGo's HTML interface, which may rate-limit or block requests if
   called too frequently.  There is no per-user rate limit on search_query
   scrape triggers beyond the general API auth requirement.  A dedicated
   rate limit for search scraping could be added if abuse becomes an issue.

2) Search result quality: DuckDuckGo results vary and some URLs may be
   paywalled, JavaScript-rendered (yielding minimal text from static HTML
   parsing), or slow to respond.  The 15-second per-URL timeout and
   graceful failure handling mitigate this, but some searches may return
   sparse or low-quality text.

3) The duckduckgo-search library is a third-party wrapper around DuckDuckGo's
   HTML search.  If DuckDuckGo changes their HTML structure, the library may
   break until updated.  Pin to 6.3.7 for stability.
